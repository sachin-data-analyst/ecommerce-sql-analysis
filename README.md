# ecommerce-sql-analysis
Ecommerce Sales Analysis using SQL
**Project Overview**
This project analyzes an eCommerce company’s sales data to uncover insights related to revenue, customer behavior, product performance, returns, and regional trends using SQL.
The goal is to simulate a real-world Data Analyst scenario, focusing on business-driven KPIs and decision-making.

**Dataset Description**
The project uses 5 relational tables:

Customers – Customer demographic details
Orders – Order-level data including return status
OrderDetails – Line-item level sales data
Products – Product catalog and pricing
Regions – Geographic segmentation

**Tools & Skills Used**
SQL (Joins, CTEs, Window Functions)
Aggregations & Conditional Logic
Time-based Analysis
Business KPI Analysis

**Key Analysis Performed**
**Sales Performance**
Total revenue and revenue excluding returns
Revenue by product and category
Monthly and yearly revenue trends

**Average Order Value (AOV)**

Overall AOV
AOV by month and region
Implemented AOV without using AVG() for accuracy

**Customer Insights**
Top 10 customers by revenue
Repeat customer rate
Customer segmentation (Platinum, Gold, Silver, Bronze)
Customer Lifetime Value (CLV)

**Returns Analysis**
Overall return rate
Return rate by product, category, and region
Identification of customers with frequent returns

**Regional Performance**
Order volume by region
Revenue and AOV by region
Best and worst performing regions

**Business Impact**
Identified high-return products impacting revenue
Highlighted top-performing regions and customers
Provided insights to improve customer retention and reduce returns
